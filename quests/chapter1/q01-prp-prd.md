# Quest 01 — プロジェクトのPRDを書く

**章**: 第1章「小さなスクリプト」  
**スキル**: `/prp-prd` を 1回  
**報酬**: 80 XP

---

## ストーリー

あなたは今、自分が作りたいプロジェクトを持っている。  
まず何を作るかを明確にしなければ、Claudeはあなたの意図を正確に理解できない。

PRD（Product Requirements Document）とは「何を作るか」を文書化したものだ。  
機能一覧ではなく、**なぜ作るか・誰のために作るか・何が成功か** を定義する。

---

## 指示

1. Claude Code のチャットで以下を実行する:

```
/prp-prd
```

2. プロンプトが表示されたら、あなたのプロジェクトについて説明する  
3. 生成されたPRDを確認し、必要なら修正する  
4. 完了したら以下を実行してXPを受け取る:

```bash
node scripts/check.js record prp-prd
```

---

## ヒント

- プロジェクトは何でも良い（CLIツール、スクリプト、ライブラリなど）
- 「完璧なPRD」は不要。まず動かすことが重要
- PRDは後から改訂できる
