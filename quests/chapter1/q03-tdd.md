# Quest 03 — TDDで開発する（3回）

**章**: 第1章「小さなスクリプト」  
**スキル**: `/tdd` を 3回  
**報酬**: 最大300 XP（初回100 + 2〜4回目×0.7）

---

## ストーリー

計画ができた。いよいよ実装だ。

しかし「とりあえずコードを書く」のは禁じ手だ。  
テスト駆動開発（TDD）では **テストを先に書き、それを通すコードを書く**。  
この順序が、のちのちの品質を決定づける。

一度ではなく **3回** 繰り返すことで、TDDのリズムが体に染みつく。

---

## 指示

機能を1つずつ実装するたびに以下を繰り返す（計3回）:

1. Claude Code のチャットで以下を実行する:

```
/tdd
```

2. テストを先に書き、それを通すコードを実装する  
3. 1サイクル完了したら以下でXPを記録する:

```bash
node scripts/check.js record tdd
```

4. 上記を計3回繰り返す

---

## ヒント

- 1サイクル = 「RED（失敗テスト）→ GREEN（通す）→ REFACTOR（整える）」
- 小さな機能から始める。完璧なテストより「何かテストがある」状態が重要
- 3回目になると自然とリズムがわかってくる
