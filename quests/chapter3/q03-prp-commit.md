# Quest 03 — コミットを作成する

**章**: 第3章「チーム開発・出荷」  
**スキル**: `/prp-commit` を 3回  
**報酬**: 約63〜90 XP

---

## ストーリー

変更の歴史は、プロジェクトのストーリーだ。

`git commit -m "fix"` や `git commit -m "update"` は、  
未来の自分や仲間に何も伝えない。  
6ヶ月後に「なぜこの変更をしたのか」を理解できるコミットメッセージを書くこと——  
これは技術スキルであり、**コミュニケーションスキル**だ。

`/prp-commit` は変更内容を分析し、  
Conventional Commits形式の意味のあるコミットメッセージを生成する。  
3回使って、「良いコミット」の感覚を身につけろ。

---

## 指示

3回のコミット作成を実施する。実際の変更をコミットするときに使う。

```
/prp-commit
```

完了後（毎回）:
```bash
node scripts/check.js record prp-commit
```

### コミットの型（Conventional Commits）

| 型 | 使いどころ |
|---|---|
| `feat:` | 新機能の追加 |
| `fix:` | バグ修正 |
| `refactor:` | リファクタリング（機能変更なし） |
| `docs:` | ドキュメント更新 |
| `test:` | テスト追加・修正 |
| `chore:` | ビルド・設定の変更 |

---

## ヒント

- コミットは「1つの論理的な変更」に絞る。複数の変更は複数のコミットに分ける
- タイトルは命令形で書く（「〜を追加した」ではなく「〜を追加する」）
- 「なぜ」この変更をしたかをbodyに書くと、将来の自分が感謝する
