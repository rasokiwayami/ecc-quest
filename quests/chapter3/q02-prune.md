# Quest 02 — 不要コードを削除する

**章**: 第3章「チーム開発・出荷」  
**スキル**: `/prune` を 3回  
**報酬**: 約126〜180 XP

---

## ストーリー

コードは書くより削る方が難しい。

「使うかもしれない」と思って残したコメントアウト、  
かつて必要だったが今は呼ばれていない関数、  
インポートしたまま使っていないライブラリ——

これらは「死んだコード」だ。  
死んだコードは読む人を惑わせ、「これは重要な何かだろうか？」と考えさせる。  
**コードベースの認知負荷を下げることが、チーム開発の速度を上げる**。

`/prune` で不要なものを取り除き、コードをクリーンに保て。

---

## 指示

3回の削除セッションを実施する。

```
/prune
```

完了後（毎回）:
```bash
node scripts/check.js record prune
```

### おすすめの削除対象（3回分）

1. **使われていないimport・require** — 未使用の依存関係
2. **コメントアウトされたコード** — 「後で使うかも」と残したコード断片
3. **呼ばれていない関数・変数** — 定義だけあって参照されていないもの

---

## ヒント

- 削除前に「本当に使われていないか」を確認する（grep や IDE の参照検索）
- 「念のため残す」は技術的負債の始まり。gitがあれば過去のコードはいつでも取り戻せる
- テストも一緒に削除対象にすること。実装を削除したらそれに対応するテストも消す
