# Quest 04 — PRを作成する

**章**: 第3章「チーム開発・出荷」  
**スキル**: `/prp-pr` を 2回  
**報酬**: 約112〜160 XP

---

## ストーリー

コードは完成した。コミットも積んだ。残るは最後の関門——**Pull Request**だ。

PRはコードを渡すだけの場ではない。  
「何を変えたか」「なぜ変えたか」「どうテストしたか」を伝える場だ。  
レビュアーが理解しやすいPRは、マージが早く、フィードバックの質も高い。

`/prp-pr` はgitの差分を分析し、意味のあるPR説明を生成する。  
2回使って、「チームに渡せるコード」の形を覚えろ。

---

## 指示

2回のPR作成を実施する。実際の変更をPRにまとめるときに使う。

```
/prp-pr
```

完了後（毎回）:
```bash
node scripts/check.js record prp-pr
```

### 良いPRの構成

```markdown
## Summary
- 何を変えたか（箇条書き3点以内）

## Why
- なぜこの変更が必要だったか

## Test plan
- [ ] どうテストしたか
- [ ] 確認すべきエッジケース
```

---

## ヒント

- PRのタイトルは70文字以内に収める
- スクリーンショットやデモGIFがあると、レビュアーの理解が格段に上がる
- 「WIP:」「Draft:」プレフィックスを使うと「まだレビュー不要」を伝えられる
