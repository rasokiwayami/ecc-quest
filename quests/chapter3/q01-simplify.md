# Quest 01 — コードをリファクタリングする

**章**: 第3章「チーム開発・出荷」  
**スキル**: `/simplify` を 3回  
**報酬**: 約105〜150 XP

---

## ストーリー

動くコードと良いコードは、違う。

第1章・第2章で書いたコードを振り返れ。  
「動いているからいい」と思って放置したコードの中に、  
読みにくい関数、重複したロジック、説明のつかない変数名が必ずある。

技術的負債は複利で増える。今日の「後でやる」は、明日の「なぜこう書いたか分からない」になる。

`/simplify` でコードを整え、**未来の自分への贈り物**を作れ。

---

## 指示

3回のリファクタリングセッションを実施する。

```
/simplify
```

完了後（毎回）:
```bash
node scripts/check.js record simplify
```

### おすすめのリファクタリング対象（3回分）

1. **長すぎる関数** — 50行を超えている関数を分割する
2. **重複コード** — 同じようなロジックが複数箇所にある部分をまとめる
3. **命名の改善** — `data`、`temp`、`result` など意味の薄い変数名を具体的にする

---

## ヒント

- リファクタリング前後でテストが通ることを確認すること（動作が変わらないことの証明）
- 「1回のセッションで1つの改善」に絞ると迷わない
- リファクタリングは機能追加ではない。コードの振る舞いを変えずに構造を整える
