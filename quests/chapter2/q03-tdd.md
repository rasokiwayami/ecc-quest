# Quest 03 — TDDでWebアプリを開発する

**章**: 第2章「Webアプリ開発」  
**スキル**: `/tdd` を 5回  
**報酬**: 約350〜500 XP

---

## ストーリー

第1章でTDDを体験した。しかし3回では「習慣」にはならない。

Webアプリのロジックはスクリプトより複雑だ。  
APIのバリデーション、データベースのCRUD操作、認証ロジック——  
テストなしで書いたコードは、変更するたびに壊れる。

**5回のTDDでサイクルを身体に刻め**。  
RED（失敗するテストを書く）→ GREEN（テストを通す）→ REFACTOR（整える）。  
このリズムが自然になったとき、あなたのコードの品質は変わる。

---

## 指示

5回のTDDセッションを実施する。毎回、新しい機能・ロジックに対して行う。

```
/tdd
```

完了後（毎回）:
```bash
node scripts/check.js record tdd
```

### おすすめのTDDテーマ（5回分）

1. **入力バリデーション** — フォームデータやAPIリクエストの検証ロジック
2. **データ変換** — DBから取得したデータを表示用に整形する関数
3. **認証ロジック** — ログイン判定・セッション管理・権限チェック
4. **エラーハンドリング** — 想定外の入力・APIエラー・DB接続失敗への対処
5. **ビジネスロジック** — アプリの核となる計算・判定・集計処理

---

## ヒント

- テストを書く前に「このテストが通ったら何が保証されるか」を1行で言語化する
- Webアプリでは「ハッピーパス」だけでなく「エラーケース」のテストが特に重要
- 5回全部を一気にやろうとしない。実装の合間に自然に使うのが正解
