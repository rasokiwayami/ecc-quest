# Quest 04 — Webアプリを実装する

**章**: 第2章「Webアプリ開発」  
**スキル**: `/prp-implement` を 3回  
**報酬**: 約240〜300 XP

---

## ストーリー

計画があり、テストがある。あとは実装するだけだ。

しかし「実装する」とはコードを書くことだけではない。  
PRDの要件を満たしているか、テストが示す動作と一致しているか、  
計画通りのフェーズで進んでいるか——**確認しながら進める**ことが重要だ。

`/prp-implement` はPRDと計画に基づいた実装を、  
確認ループを挟みながら進めてくれる。  
3回使って、「設計通りに実装する」感覚を掴め。

---

## 指示

3回の実装セッションを実施する。フェーズや機能ごとに分けて行う。

```
/prp-implement
```

完了後（毎回）:
```bash
node scripts/check.js record prp-implement
```

### おすすめの実装の分け方（3回分）

1. **コア機能** — アプリの中心となる機能（CRUD操作、メイン画面など）
2. **補完機能** — 認証、バリデーション、エラー処理
3. **仕上げ** — UI調整、パフォーマンス改善、エッジケース対応

---

## ヒント

- 1回の実装セッションを「1機能」に絞ると集中しやすい
- 実装前に「このセッションのゴールは何か」を1行で定義する
- `/tdd` で書いたテストがあれば、それを実装の検証基準として使う
