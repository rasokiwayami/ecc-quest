# Quest 05 — コードレビューを受ける

**章**: 第2章「Webアプリ開発」  
**スキル**: `/code-review` を 3回  
**報酬**: 約108〜144 XP

---

## ストーリー

コードは動いている。しかしそれで十分か？

自分が書いたコードは自分では見えない部分がある。  
命名の曖昧さ、関数の肥大化、重複ロジック、将来のバグの種——  
書いた本人には「当たり前」に見えて、他者には「なぜ？」と見えるコードが必ずある。

`/code-review` は経験豊富なレビュアーの目線でコードを見る。  
3回使って、「自分では気づけなかった問題」を発見する体験を積め。

---

## 指示

3回のコードレビューセッションを実施する。異なるファイルや機能に対して行う。

```
/code-review
```

完了後（毎回）:
```bash
node scripts/check.js record code-review
```

### おすすめのレビュー対象（3回分）

1. **最も複雑な関数・モジュール** — ロジックが集中している箇所
2. **データの入出力処理** — バリデーション、変換、フォーマット処理
3. **全体の構造レビュー** — ファイル構成・命名・依存関係の整合性

---

## ヒント

- CRITICAL（致命的）指摘は必ず修正すること
- HIGH（重要）指摘はできれば修正し、理由を記録する
- レビュー後に「なぜそう指摘されたのか」を1行で自分の言葉で書いておくと記憶に残る
