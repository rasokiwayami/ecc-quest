# Quest 06 — セキュリティレビュー

**章**: 第2章「Webアプリ開発」  
**スキル**: `/security-review` を 2回  
**報酬**: 約84〜96 XP

---

## ストーリー

Webアプリはインターネットに接続する。それは、世界中からアクセスされる可能性があるということだ。

SQLインジェクション、XSS、認証バイパス、APIキーの漏洩——  
Webの世界にはスクリプトにはなかった脅威が存在する。  
**セキュリティの問題は、発覚したときには手遅れなことが多い**。

`/security-review` は攻撃者の視点でコードを検査する。  
2回使って、「セキュリティを意識して書く」目線を身につけろ。

---

## 指示

2回のセキュリティレビューセッションを実施する。

```
/security-review
```

完了後（毎回）:
```bash
node scripts/check.js record security-review
```

### おすすめのレビュー対象（2回分）

1. **認証・認可コード** — ログイン処理、セッション管理、権限チェック
2. **外部入力を扱うコード** — フォーム、APIリクエスト、ファイルアップロード

---

## ヒント

- OWASP Top 10 を一度だけ読んでおくと、レビュー結果の意味が深く理解できる
- 環境変数・シークレットの扱いは必ず確認する（ハードコードは即修正）
- 「自分のアプリには関係ない」と思った指摘こそ、実は重要なことが多い
