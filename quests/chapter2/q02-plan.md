# Quest 02 — Webアプリの計画を立てる

**章**: 第2章「Webアプリ開発」  
**スキル**: `/plan` を 2回  
**報酬**: 約112 XP

---

## ストーリー

PRDで「何を作るか」が決まった。しかし、Webアプリは複雑だ。

フロントエンドのコンポーネント設計、バックエンドのAPI設計、  
データベースのスキーマ設計——これらが噛み合って初めてアプリは動く。

スクリプトなら「上から下に書けば動く」。  
Webアプリはそうはいかない。**計画なしに進むと、必ずどこかで作り直しが発生する**。

2回の計画立案で、「構造的に考える」習慣を身につけろ。

---

## 指示

**1回目**: Webアプリ全体のアーキテクチャ計画を立てる

```
/plan
```

PRDをもとに、アプリ全体の実装計画を作成する。  
フェーズ分け（「まず動くもの」→「機能を整える」→「デプロイ」）を意識する。

完了後:
```bash
node scripts/check.js record plan
```

**2回目**: 特定のコンポーネントや機能の詳細計画を立てる

```
/plan
```

1回目の計画のうち、最も複雑な部分を選んでさらに深掘りする。  
例: 「認証機能の詳細設計」「データベーススキーマの設計」

完了後:
```bash
node scripts/check.js record plan
```

---

## ヒント

- 2回目の計画は「粒度を下げる」のがポイント。1回目はフェーズ、2回目はタスクレベルまで落とす
- 「この計画のリスクは何か」を必ず1つ以上挙げること
- 技術スタックが決まっていない場合は、計画の中で決めてしまうと後が楽
