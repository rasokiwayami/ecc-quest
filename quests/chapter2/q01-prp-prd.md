# Quest 01 — WebアプリのPRDを書く

**章**: 第2章「Webアプリ開発」  
**スキル**: `/prp-prd` を 2回  
**報酬**: 約112 XP

---

## ストーリー

第1章のスクリプトは動いた。あなたは「作れる」ことを証明した。

次のステージはWebアプリだ。スクリプトと違い、Webアプリには複数の層がある——  
フロントエンド、バックエンド、データベース、API。  
規模が大きくなるほど、**最初の設計がすべてを決める**。

適当に書き始めたWebアプリは、必ず途中で壊れる。  
PRDを2回書くことで、「大きなものを設計する」感覚を掴め。

---

## 指示

**1回目**: 第2章で取り組むWebアプリのPRDを書く

```
/prp-prd
```

第1章で選んだプロジェクトをWebアプリに発展させても良いし、  
まったく新しいWebアプリを定義しても良い。

完了後:
```bash
node scripts/check.js record prp-prd
```

**2回目**: PRDを改訂するか、別の機能・別のプロジェクトのPRDを書く

```
/prp-prd
```

1回目のPRDを読み返し、曖昧だった部分を明確にする。  
または、アプリの「管理画面」「API仕様」など特定の領域にフォーカスしたPRDを追加する。

完了後:
```bash
node scripts/check.js record prp-prd
```

---

## ヒント

- 2回目は「より深く」書くことを意識する。ユーザーストーリー・受け入れ条件・非機能要件を加えてみる
- Webアプリに適したPRDには「技術スタック」「デプロイ先」「セキュリティ要件」の項目が入ることが多い
- 「何を作らないか（スコープ外）」を明示すると質が一段上がる
