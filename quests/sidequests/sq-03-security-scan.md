# Side Quest 03 — セキュリティスキャン

**章**: サイドクエスト「探索と深化」  
**スキル**: `/security-scan` を 3回  
**報酬**: 約126〜180 XP  
**解放条件**: いつでも挑戦可能

---

## ストーリー

セキュリティは一度チェックして終わりではない。

コードは成長する。新しい機能が追加されるたびに、新しい脆弱性が生まれる可能性がある。  
ハードコードされたAPIキー、安全でない依存パッケージ、設定ミス——  
これらは「書いたとき」には気づかず、「運用中」に発覚することが多い。

`/security-scan` はコードベース全体を静的解析し、潜在的なリスクを洗い出す。  
3回使って、**継続的にセキュリティを意識する**習慣を身につけろ。

---

## 指示

3回のセキュリティスキャンを実施する。実装が進むたびに定期的に行う。

```
/security-scan
```

完了後（毎回）:
```bash
node scripts/check.js record security-scan
```

### スキャンのタイミング

1. **新機能追加後** — 実装直後にスキャン
2. **依存パッケージ更新後** — npm install や package.json 変更後
3. **デプロイ前** — 本番環境に出す前の最終確認

---

## ヒント

- スキャン結果の全指摘を一度に修正しようとしない。CRITICAL → HIGH → MEDIUM の順に対処する
- 「false positive（誤検知）」もある。なぜその指摘が出たかを理解することが重要
- スキャンは「問題を見つける」ためではなく「問題がないことを確認する」ためでもある
