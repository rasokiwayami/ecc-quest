# Side Quest 01 — 深いリサーチ

**章**: サイドクエスト「探索と深化」  
**スキル**: `/deep-research` を 2回  
**報酬**: 約200 XP  
**解放条件**: いつでも挑戦可能

---

## ストーリー

実装の前に、立ち止まって調べることができるか。

「とりあえず書いてみる」で始まったコードが、  
後から「実はこういうライブラリがあった」「このパターンが一般的だった」と分かる——  
そういう経験をしたことがあるはずだ。

リサーチは「時間の無駄」ではない。  
深く調べることで、**作らなくて良いものを作らずに済む**。  
`/deep-research` で、「よく調べた上で作る」習慣を手に入れろ。

---

## 指示

2回のリサーチセッションを実施する。

```
/deep-research
```

完了後（毎回）:
```bash
node scripts/check.js record deep-research
```

### おすすめのリサーチテーマ

- 使おうとしているライブラリの比較検討（例: 認証ライブラリはどれが良いか）
- 採用しようとしているアーキテクチャパターンの事例調査
- 似たようなツール・サービスがすでに存在するかの調査
- セキュリティベストプラクティスの調査

---

## ヒント

- リサーチの結果を `docs/` フォルダに残すと、将来の意思決定の根拠になる
- 「なぜこの選択をしたか」を記録することが、チーム開発では特に重要
- リサーチは「答えを見つける」ではなく「良い問いを立てる」プロセス
