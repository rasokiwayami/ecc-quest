# Side Quest 02 — ドキュメントを書く

**章**: サイドクエスト「探索と深化」  
**スキル**: `/docs` を 3回  
**報酬**: 約105〜150 XP  
**解放条件**: いつでも挑戦可能

---

## ストーリー

コードは書いた人にしか読めない——そう思っていないか？

READMEのないリポジトリは、地図のない迷宮だ。  
「セットアップ方法がわからない」「どのAPIを呼べばいいのか」——  
ドキュメントがなければ、優れたコードも使われない。

ドキュメントはコードを人間の言葉に翻訳する行為だ。  
書くことで自分の理解も深まる。  
`/docs` で、**伝わるコード**を作れ。

---

## 指示

3回のドキュメント作成セッションを実施する。

```
/docs
```

完了後（毎回）:
```bash
node scripts/check.js record docs
```

### おすすめのドキュメント対象（3回分）

1. **README.md** — プロジェクトの概要・セットアップ手順・使い方
2. **API仕様** — エンドポイント一覧・リクエスト/レスポンス形式
3. **アーキテクチャ説明** — ディレクトリ構成・主要コンポーネントの役割

---

## ヒント

- 「初めてこのプロジェクトを見た人」を読者として想定して書く
- コードブロックとサンプルコマンドを多用すると格段に読みやすくなる
- ドキュメントは完璧である必要はない。「ないよりあった方がいい」が正解
